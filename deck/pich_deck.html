01 / PROBLEM & SOLUTION

Solving the core risks of AI-agent trading

⚠️ The Problem

  • Manual approval doesn't scale: on-chain opportunities fire 24/7, but waking up to approve every action means missing them.
  • Handing over the keys is terrifying: give an agent your funds directly and a bug, a hack, or "a wrong intent optimized perfectly" can wipe you out.
  • The infra wall: safely building an isolated runtime — and depositing funds and keys into it — is extremely hard to get right.

🚀 The Solution

  • True self-custody via EIP-7702: funds never leave your wallet — they stay in your own EOA while the agent trades within bounds.
  • Mechanical × semantic guardrails: the contract (Hard) rejects invalid trades, while a separate Watcher agent reads market context and auto-tightens the limits.
  • Agents as NFTs (ERC-721/8004): intent and execution identity are encapsulated as an NFT — the foundation of an autonomous agent economy.
💡 Core idea: you don't have to fear a malicious or buggy AI — by design, it structurally cannot act outside the boundary the owner drew.

02 / ARCHITECTURE

Robustness through EIP-7702 × role separation (Executor / Watcher)

Layer 01 / Intelligence

🧠 Executor Agent

Reasons about trading strategy in an isolated runtime (Cloud Run). It only signs execution requests with a GCP KMS session key — it never holds funds.

Layer 02 / Authority

⚡ EIP-7702 Contract

Execution code delegated onto the owner's EOA. It enforces Hard Guardrails on-chain and instantly reverts any request outside the preset limits.

Layer 03 / Monitoring

👁️ Watcher Agent

Continuously monitors the market and execution logs. A one-way governance design: on anomalies it can only tighten or freeze the limits — never loosen them.
